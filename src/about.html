SchedulerFiles

Utility Java per copiare, spostare, confrontare e organizzare file.

Guida rapida

  1. Seleziona Source: cartella origine. Usa campo, Browse o drag & drop.
  2. Seleziona Dest: prima destinazione. Cartella viene creata se necessario.
  3. Opzionale: inserisci Dest 2. In modalità Copy, file vengono replicati su entrambe destinazioni, come RAID-1 software.
  4. Scegli Copy per conservare origine oppure disattivalo per Move.
  5. Scegli struttura:
    • Original Tree: mantiene gerarchia origine.
    • Scheduled: organizza per anno/mese/estensione.
  6. Premi Start. Usa Cancel per interrompere operazione.

Confronto e integrità

  • Compare by content: evita duplicati; usa SHA-256 quando verifica hash attiva.
  • Compare by name: evita file con stesso nome.
  • Verify SHA-256: attivo di default; verifica contenuto prima/dopo trasferimento.

Dest 2 / RAID-1 software

Dest 2 è opzionale. Se compilato, Copy esegue prima trasferimento su Dest, poi replica su Dest 2. Operazione termina con errore se una destinazione fallisce. Dest 2 deve essere diversa da Source e Dest. Move con Dest 2 non è consentito.

Se Dest 2 contiene file extra, Info chiede se eliminarli, conservarli o annullare. Replica non è RAID hardware né transazione atomica: se secondo disco si guasta, controlla log e file mancanti prima di riprovare.

Log e sicurezza

_files_scheduler.log viene scritto in ogni destinazione. Contiene operazioni, timestamp, dimensioni e hash.

Fai backup prima di operazioni Move. Software fornito “as is”, senza garanzia.
